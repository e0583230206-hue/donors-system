מסמך זה נגזר ישירות מהקוד הקיים (`ivr.js`, `ivr.service.js`, `donor.service.js`, `server.js`, `donor.html`) ומהחלטות שאושרו בסבב תכנון מלא. **אין בו תרגום לאידיש, אין הקלטה בפועל, ואין שינוי קוד.** נעול סופית לשינויי מבנה — כל שינוי נוסף ייפתח כגרסה 1.1.

## 4. מבנה תיקיות + Naming Convention (ללא שינוי)

```
ivr-audio/
├── manifest.json
├── packs/
│   └── yi-default/
│       ├── open/ menu/ debt/ pay/ voicemail/ system/ purpose/
│       └── numbers/{digits,teens,tens,hundreds,thousands,currency}/
├── he-reference/
└── qa/
```

שם קובץ: `ivr_<category>_<id>_<lang>_v<version>.wav`

*(תיקיית `greeting/` הוסרה — GRT-002 בוטל, אין עוד קטגוריית ברכה נפרדת.)*

## 5. Manifest — שדות סופיים (v1.0)

```json
{
  "id": "DEBT-001",
  "category": "debt",
  "referenceTextHe": "יש לך חוב על סכום",
  "pack": "yi-default",
  "version": 1,
  "status": "draft",
  "file": "packs/yi-default/debt/ivr_debt_001_yi_v1.wav",
  "durationMs": null,
  "notes": "",
  "updatedAt": null
}
```
`status`: draft → approved → production. המערכת החיה משמיעה רק `production`; כל דבר אחר נופל לשרשרת ה-Fallback.

## 6. שרשרת Fallback (ללא שינוי)

חסר קובץ / לא `production` → גרסה קודמת של אותו ID → חבילת ברירת מחדל → טקסט עברי קיים (`T` ב-`ivr.js`, TTS) → הודעת שגיאה גנרית + ניתוק.

## 7. מה עדיין פתוח מול טכנוליין (לא תלוי במתרגם, לא חוסם שליחה)

1. תמיכת אידיש במודול האשראי הפנימי (`type:"creditCard"`).
2. פורמט קובץ שמע נדרש (WAV/דגימה/ערוץ).
3. מנגנון הפניה לקובץ — URL ציבורי מול העלאה מראש עם מזהה פנימי.
4. התנהגות Cache שלהם מול ה-URL המבוסס-גרסה שתכננו.
5. CREDIT_CARD_TERMINAL — פריט ישן שעדיין ממתין להם, לא קשור להקלטות.
